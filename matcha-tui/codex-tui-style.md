# Codex TUI: заметки по стилю и как перенести в matcha-tui

Ниже — разбор визуальных и поведенческих элементов Codex TUI по коду и практические рекомендации, как перенести этот стиль в текущий Ink‑интерфейс.

## Что я посмотрел в коде Codex

- `codex-rs/cli/src/main.rs` — выбор между `codex_tui` и `codex_tui2` по фиче `tui2`.
- `codex-rs/tui/src/history_cell.rs` — заголовочная карточка, рамки и функция `with_border` с box‑drawing.
- `codex-rs/tui/src/ui_consts.rs` — единый левый гаттер `LIVE_PREFIX_COLS = 2`.
- `codex-rs/tui/src/bottom_pane/footer.rs` — футер: короткая строка подсказок, разделители `·`, dim‑стиль.
- `codex-rs/tui/src/key_hint.rs` — формат клавиш (нижний регистр, стрелки, dim).
- `codex-rs/tui/src/status_indicator_widget.rs` — статус‑строка со спиннером, временем и подсказкой `Esc`.
- `codex-rs/tui/src/style.rs`, `terminal_palette.rs`, `color.rs` — адаптация цветов под терминал и мягкий фон для user‑сообщений.
- `codex-rs/tui/src/onboarding/welcome.rs` — минималистичный welcome‑экран.

## Принципы стиля, которые дают «аккуратно и компактно»

1. **Минимум визуального шума.** Рамки используются точечно и всегда приглушены (dim), чтобы не «забивать» контент.
2. **Одна колонка + стабильный левый гаттер.** Весь интерфейс визуально выровнен по левой линии шириной 2 колонки.
3. **Компактная вертикаль.** Между секциями — максимум одна пустая строка.
4. **Цвет как акцент, не как фон.** Основной текст — нейтральный, акценты — cyan/white, вторичное — dim/gray.
5. **Короткие строки состояния.** Важные статусы укладываются в 1 строку, детали — во 2–3 строки максимум.

## Компоненты, формирующие стиль

### 1) Заголовочная карточка (header card)

Ключевой элемент: небольшая карточка с рамкой и текстом внутри.

- В `history_cell.rs` функция `with_border` рисует рамку символами `╭─╮│╰─╯`.
- Ширина карточки ограничена `SESSION_HEADER_MAX_INNER_WIDTH = 56` — это делает блок компактным.
- Внутри 3–4 строки: заголовок `>_ OpenAI Codex (vX)`, затем модель и директория.

**Идея для переноса:** заменить громоздкий баннер на «карточку» в 3–4 строки и ограничить ширину.

### 2) Футер с подсказками и статусом

`bottom_pane/footer.rs`:

- Одна строка подсказки внизу экрана.
- Формат: `context · ? for shortcuts` (используется разделитель `·`).
- Все подсказки и клавиши — **dim**.

**Идея для переноса:** сделать футер одной строкой, где ключи и подсказки выглядят как вторичный текст.

### 3) Формат клавиш (key hints)

`key_hint.rs`:

- Клавиши выводятся в нижнем регистре (`enter`, `pgup`, `esc`).
- Стрелки рисуются символами `↑↓←→`.
- Все key‑hints — **dim**.

**Идея для переноса:** в Ink‑футере показывать `[esc]`, `[enter]` в едином стиле и цвете.

### 4) Строка статуса задачи

`status_indicator_widget.rs`:

- Формат: `spinner + "Working" + (elapsed • Esc to interrupt)`.
- Подсказка `Esc` — dim.
- Детали задачи выводятся ниже, с префиксом `└`, максимум 3 строки.

**Идея для переноса:** в Running‑экране заменить «многострочный» хедер на компактную строку:  
`⠋ Docking (1m 03s • Esc to interrupt)`.

### 5) Цвета и фон

`style.rs` + `terminal_palette.rs` + `color.rs`:

- Цвета подбираются под возможности терминала (16m/256/16).
- Фон user‑сообщения — это **легкий 10% blend** с фоном терминала, чтобы не создавать тяжелые блоки.

**Идея для переноса:** не красить большие блоки; вместо этого использовать dim/gray и 1–2 акцентных цвета.

## Конкретные рекомендации для matcha-tui

### 1) Уведомления в стиле Codex

Сейчас уведомление рендерится отдельным блоком в `App.tsx`. Для «тихого» вида:

- Перенести отображение уведомления в `Footer`.
- Рендерить как dim/gray одну строку.

Пример (Ink):

```tsx
const notification = useStore((s) => s.notification);

{notification && (
  <Box paddingX={1}>
    <Text color="gray" dimColor>{notification}</Text>
  </Box>
)}
```

Чтобы показать строку вроде `changed 1 package in 3s`:

```tsx
const setNotification = useStore((s) => s.setNotification);
setNotification("changed 1 package in 3s");
```

### 2) Уплотнить хедер

Сделать «карточку» вместо большого ASCII‑лого:

- Верхняя строка: `>_ MATCHA TUI (vX)`.
- Ниже: `model:` и `directory:` (как у Codex).
- Рамку рисовать символами box‑drawing и dim‑цветом.

### 3) Футер в одну строку

Вместо списка кнопок — одна строка вида:

```
context: 12% · ? for shortcuts · esc to back
```

Это сразу делает UI «тихим» и компактным.

### 4) Running‑экран как статус‑строка

Сейчас в `Running.tsx` много отдельных секций. Можно:

- Оставить основной прогресс, но заменить «Elapsed»+лиганды на одну строку.
- Вынести `Esc` или `c` подсказки в футер, чтобы верх был чище.

## Мини‑чеклист визуальной аккуратности

- Рамки только там, где они *добавляют смысл*.
- Второстепенная инфа всегда dim/gray.
- Одна строка — один смысл.
- Разделители `·` вместо длинных списков.
- Единый левый гаттер.
